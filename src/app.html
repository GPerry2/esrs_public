Core uses gulp-preprocess to preprocess HTML, Javascript, and CSS files. Here are some ways to use it:

  • Generate a url to a project resource, for example an image: /img/sample.jpg
  • Print out preprocessor values. Here are the ones included in every core project (you can add more in your gulpfile):
    • ENV is the current build/run environment:
    • SRC_PATH is the deployment path to the src folder:
    • BUILD_ID is a unique id/timestamp generated every time you build the app:
    • QA_RELEASE is the current value of the qaRelease property of your package.json file:
    • VERSION is the current value of the version property of your package.json file:
    • IS_EMBEDDED indicates whether or not this is an embedded app:
  • Include code based on the current build environment: This sentence, for example, is only visible when running on your local machine or on a development build.
  • You can do more advanced things as well. See preprocess for more about preprocessor syntax.
